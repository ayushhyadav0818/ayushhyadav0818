## ◈ Featured Projects

<details>
<summary><b>⬡ TradeBook — Enterprise Inventory Management System</b></summary>

<br/>

A production-grade desktop inventory and trade management system built on Java Swing with a JDBC-connected relational database backend. Designed for small-to-mid scale retail operations requiring real-time stock tracking, vendor management, and transaction history.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | Java · Java Swing · JDBC · MySQL · OOP · MVC |
| **Scale** | Desktop application — multi-table relational schema |
| **Performance** | Sub-100ms query response via optimized JDBC connection pooling |
| **Security** | Role-based access; parameterized queries against SQL injection |
| **Impact** | Full CRUD lifecycle for inventory, vendors, and transactions |
| **Repository** | [github.com/ayushhyadav0818](https://github.com/ayushhyadav0818) |

</div>

TradeBook was architected with a strict separation of data, business logic, and presentation layers. The backend uses a normalized MySQL schema with foreign key constraints to enforce referential integrity across inventory, supplier, and transaction tables. The UI layer in Java Swing was built to minimize repaints and handle event-driven state transitions without blocking the EDT. This project served as the primary demonstration of Java backend engineering on the portfolio and was built entirely without frameworks — every component is hand-implemented.

</details>

<details>
<summary><b>⬡ AI-Powered Design Search Engine — LLM + API Orchestration CLI</b></summary>

<br/>

A Python-native CLI application that combines the Dribbble REST API with Claude AI to enable natural-language semantic search over design assets. Features a Rich terminal interface, intelligent query reformulation, local SQLite caching, and Google Antigravity SDK integration.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | Python · Claude API · Dribbble REST API · Google Antigravity SDK · SQLite · Rich |
| **Scale** | API-orchestrated — multi-source retrieval with local fallback |
| **Performance** | Local DB cache reduces redundant API calls by ~70% on repeated queries |
| **Security** | API key isolation via environment variables; rate-limit-aware request queuing |
| **Impact** | Natural language → design asset retrieval; runs fully in terminal |
| **Repository** | [github.com/ayushhyadav0818](https://github.com/ayushhyadav0818) |

</div>

This project was built to demonstrate a real production integration pattern: an AI model (Claude) sits as an orchestration layer between a user's natural language query and a structured third-party API. The engine reformulates queries using Claude's reasoning capabilities, fetches semantically relevant results from Dribbble, and falls back to a local SQLite database when the API is unavailable. The Rich CLI provides a polished terminal UX with color-coded result cards, pagination, and query history.

</details>

<details>
<summary><b>⬡ Resume Intelligence Engine — AI-Powered ATS Analyzer</b></summary>

<br/>

A browser-based resume analysis tool that uses the Claude API to evaluate uploaded resumes against ATS (Applicant Tracking System) criteria, returning structured scores across impact, keyword density, and formatting quality.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | HTML · CSS · JavaScript · Claude API (claude-sonnet-4-6) |
| **Scale** | Client-side SPA — zero-backend, API-direct architecture |
| **Performance** | Streaming Claude responses; real-time score rendering |
| **Security** | No server-side storage; PDF parsed and analyzed in-browser |
| **Impact** | ATS score · Impact score · Keyword score · Actionable recommendations |
| **Repository** | [github.com/ayushhyadav0818](https://github.com/ayushhyadav0818) |

</div>

This project demonstrates a zero-backend AI product pattern: the Claude API is called directly from the browser with a structured prompt schema, and the response is streamed into a real-time results view with three scored dimensions — ATS compliance, impact framing, and keyword optimization. Each score returns specific, actionable recommendations. The UI is designed for recruiter-adjacent use cases and was built as the MVP for a broader resume optimization SaaS concept.

</details>

<details>
<summary><b>⬡ Jan Awaaz — Civic Tech Grievance Platform (Concept / Architecture)</b></summary>

<br/>

A two-sided civic technology platform for India, architected to enable citizens to file and track public grievances with SLA-based auto-escalation, tamper-evident audit logs, and authority performance scoring.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | Next.js 14 · Supabase · MSG91 · Resend · Vercel |
| **Scale** | National-scale — district, state, and central authority hierarchy |
| **Performance** | Edge-deployed; real-time SLA countdown with Supabase triggers |
| **Security** | Tamper-evident audit logs; corruption-bypass routing logic |
| **Impact** | Public authority directory · SLA escalation · Citizen-facing transparency |
| **Repository** | [github.com/ayushhyadav0818](https://github.com/ayushhyadav0818) |

</div>

Jan Awaaz (जन आवाज़) was designed as a govtech product to address the structural accountability gap in India's public grievance systems. The platform architecture features a two-sided model: citizens submit and track complaints with live SLA timers, while authority dashboards display resolution rates and performance scores. Auto-escalation routes unresolved tickets up the authority chain, and all state transitions are logged in an append-only audit table. Built as a solo developer project with a master AI build prompt for deployment via Cursor, Bolt, and v0.

</details>
